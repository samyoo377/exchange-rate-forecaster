# 产品侧的初步思路（优化版）

## 一、背景
汇率预测项目一期目标是构建“可运行、可解释、可复盘”的预测工作台，用于为研究与策略制定提供辅助判断。项目聚焦 USD/CNH（日线为主，支持后续扩展）。

需要明确：
- 外汇预测不存在绝对准确模型，系统输出仅作为辅助决策依据。
- 技术指标在重大政治/金融事件冲击下可能短时失效。
- 一期以“先跑通闭环”为核心：数据抓取 → 指标计算 → 信号汇总 → 预测输出 → 历史沉淀。

## 二、方案概要
### 2.1 一期目标
1. 建立统一数据入口：接入 Alpha Vantage 获取 OHLC。
2. 建立统一指标层：完成核心震荡/趋势指标计算。
3. 建立统一输出层：将预测结果结构化输出到页面与历史记录。
4. 建立可追溯能力：每次预测均关联数据版本、时间戳和指标快照。

### 2.2 一期边界
- 范围内：单币对（USDCNH）、日线级别、规则驱动预测、页面展示与历史查询。
- 范围外：自动交易执行、多币对组合回测、事件驱动高频预测。

### 2.3 原则
- 先稳定后复杂：优先实现可维护、可验证版本。
- 先结构化后智能化：先固化数据与输出 schema，再逐步增强模型。
- 先可解释后高精度：确保每条结论可解释、可复盘。

## 三、详细方案
## 3.1 数据获取
### 3.1.1 数据来源
- 主数据源：Alpha Vantage（https://www.alphavantage.co/documentation/）
- 数据类型：OHLC（Open/High/Low/Close）
- 推荐粒度：Daily
- 备选路径：本地 Excel 导入（用于离线验证和回退）

### 3.1.2 数据字段口径
对每个交易日保留以下字段：
- symbol：币对，默认 USDCNH
- timestamp：交易日
- open：开盘价
- high：最高价
- low：最低价
- close：收盘价
- source：数据来源
- version：数据版本号

### 3.1.3 数据质量规则
- high >= low
- high >= open/close
- low <= open/close
- 同一 symbol + timestamp 不可重复写入
- 缺失值/异常值记录为失败任务并写入日志

## 3.2 数据指标体系
一期先实现五类核心指标，形成可解释信号层。

### 3.2.1 RSI(14) 相对强弱指标
计算逻辑：
1. 取最近 14 个周期收盘价。
2. 计算逐日涨跌幅，分离上涨幅度与下跌幅度。
3. 计算平均涨幅与平均跌幅。
4. RS = 平均涨幅 / 平均跌幅。
5. RSI = 100 - [100 / (1 + RS)]。

信号口径：
- 买入倾向：RSI < 30 且 RSI 当期上拐
- 卖出倾向：RSI > 70 且 RSI 当期下拐
- 中立：其余情况

### 3.2.2 Stochastic %K(14,3,3)
核心公式：
- 原始%K = 100 × (当前收盘价 - 14期最低价) / (14期最高价 - 14期最低价)
- 最终%K = 最近 3 期原始%K 均值
- %D = 最近 3 期最终%K 均值

信号口径：
- %K > 80：卖出倾向
- %K < 20：买入倾向
- 其余：中立

### 3.2.3 CCI(20) 顺势指标
核心公式：
1. TP = (High + Low + Close) / 3
2. MA = TP 的 20 期均值
3. MD = 平均偏差
4. CCI = (TP - MA) / (0.015 × MD)

信号口径：
- CCI > 100：卖出倾向
- CCI < -100：买入倾向
- 其余：中立

### 3.2.4 ADX(14) 平均趋向指标
核心目的：判断趋势强弱而非直接方向。

信号口径：
- ADX 0~20（或 25）：弱趋势/盘整
- ADX 20/25~50：趋势增强
- ADX 50~100：极强趋势

解释规则：
- 若 ADX 高且 +DI > -DI，则多头趋势可信度更高。
- 若 ADX 高且 -DI > +DI，则空头趋势可信度更高。

### 3.2.5 AO 动量震荡指标
核心公式：
1. 中位数价格 MP = (High + Low) / 2
2. AO = SMA(MP,5) - SMA(MP,34)

信号口径（一期简化）：
- AO 连续上升且 > 0：偏多
- AO 连续下降且 < 0：偏空
- 其余：中立

### 3.2.6 Momentum(10) 动量指标（补全）
建议采用标准动量定义：
- MOM(10) = Close(t) - Close(t-10)

信号口径：
- MOM > 0：偏多
- MOM < 0：偏空
- MOM 绝对值接近 0：中立

## 3.3 信号融合与预测结论
### 3.3.1 融合方法（一期）
采用“规则加权打分”而非复杂模型：
- RSI、Stochastic、CCI、AO、MOM：各自输出 buy/sell/neutral
- ADX：作为趋势强度修正因子

示例：
- buy = +1，sell = -1，neutral = 0
- 指标总分 = 各指标分值求和
- ADX > 25 时，趋势方向分值乘以 1.2

### 3.3.2 输出口径
最终输出字段：
- direction：bullish / bearish / neutral
- confidence：0~1
- rationale：关键依据数组
- riskNotes：风险提示数组
- sourceRefs：数据来源
- generatedAt：生成时间
- snapshotVersion：数据版本

### 3.3.3 风险提示口径
每条输出必须包含至少一条风险提示：
- 技术指标仅基于历史价格行为
- 突发事件可能导致信号失效
- 预测仅作策略辅助，非交易建议

## 3.4 页面输出 Demo（文字版）
### 3.4.1 工作台布局
- 顶部：币对、更新时间、刷新按钮
- 左区：聊天问答（输入问题、展示预测）
- 右区：K 线/趋势图、指标卡片、最新结论
- 底部：历史预测记录列表

### 3.4.2 预测卡片模板
- 结论：未来 T+2 偏升/偏贬/震荡
- 置信度：xx%
- 关键依据：
  - RSI 从超卖区回升
  - AO 由负转正
  - ADX 提升至趋势区间
- 风险提示：事件冲击风险、数据滞后风险

## 四、任务拆解（产品侧）
### 4.1 P0（一期必须）
1. 数据拉取链路打通（Alpha + 本地导入）
2. 五类核心指标计算
3. 规则融合输出结构化结论
4. 页面展示（聊天 + 大盘 + 历史）
5. 数据版本与任务日志可追溯

### 4.2 P1（一期增强）
1. 多币对支持
2. 指标参数可配置化
3. 预测结果导出（JSON/CSV）
4. 中台结果筛选与对比

### 4.3 P2（后续演进）
1. 加入宏观数据与新闻事件标签
2. 规则 + 机器学习混合建模
3. 回测与胜率跟踪
4. 多策略版本管理

## 五、验收标准
### 5.1 功能验收
- 可成功拉取并入库最近行情数据。
- 可计算并展示 RSI、Stochastic、CCI、ADX、AO、MOM。
- 可基于规则输出结构化预测结果。
- 可在页面查看最新结论与历史记录。

### 5.2 数据验收
- 指标计算结果可复算。
- 每条预测可回溯到数据时间戳和版本。
- 异常数据和失败任务有日志记录。

### 5.3 业务验收
- 输出中包含“结论 + 依据 + 风险”。
- 明确展示“预测非交易建议”。
- 支持 T+2 预测口径。

## 六、开放问题与待确认
1. 一期是否仅限 USDCNH。
2. 预测周期默认 T+2，是否需要 T+1/T+3 可选。
3. 历史数据最少保留天数与归档策略。
4. 中台是否需要角色权限（查看/管理）。
5. 输出导出优先 JSON 还是 Excel。

## 七、版本结论
本方案以“可解释、可追溯、可扩展”为核心，先实现规则驱动的最小预测闭环，再逐步增加因子、增强模型和提高稳定性。该路径可在控制工程复杂度的前提下，快速形成可用产品能力。
