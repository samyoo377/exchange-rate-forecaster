# 汇率预测机PRD

## 1. 文档目标
本文档用于定义“汇率预测机”产品的一期目标、用户场景、功能边界、核心流程、验收标准与实施优先级，为后续设计、研发、测试与运营提供统一依据。

## 2. 项目背景
结合现有资料，项目的初步目标包括三部分：
1. 提供一个 AI 预测机的承载页面，包含聊天界面与数据大盘
2. 提供一个能够消化业务数据并进行策略演算的 Agent
3. 提供一个能够自动扒取数据、总结并结构化输出结果的 Agent，且输出可在文件与中台中查看

现有代码库仍处于非常早期阶段：
- package.json 仅包含基础 Node 项目信息，尚未形成完整前端或服务端工程结构
- api.ts 当前仅出现 Alpha Vantage 文档地址，说明外汇数据接入方向初步指向第三方金融数据源
- 本地还存在一个 PDF 项目材料与一个 20 日美元兑人民币数据表，说明后续可结合样例数据与业务资料推进指标口径、展示方式与验证逻辑

## 3. 产品愿景
打造一个面向业务分析、研究判断与策略辅助决策的“汇率预测机”工作台，使用户能够在一个统一界面中完成：
- 查看汇率市场数据与关键指标
- 与 AI 进行趋势问答与预测交互
- 获取基于数据和策略演算的预测结果与解释
- 跟踪数据来源、结构化结论和策略输出

## 4. 目标用户
### 4.1 核心用户
- 业务分析人员
- 外汇研究人员
- 交易策略辅助人员
- 管理中台使用者

### 4.2 次级用户
- 产品经理
- 数据运营人员
- 需要查看日报、周报或结构化摘要的管理者

## 5. 一期产品目标
### 5.1 业务目标
- 建立汇率预测的统一工作入口
- 形成“数据抓取 → 数据处理 → 策略演算 → 预测输出 → 页面展示”的闭环
- 让中台人员可查看原始数据摘要、结构化结论与预测建议

### 5.2 产品目标
- 完成聊天界面与数据大盘的一体化页面
- 完成两个核心 Agent 的职责拆分与协同
- 支持至少一种外部汇率数据源接入
- 支持预测结果的结构化展示与可追溯说明

### 5.3 非目标
一期暂不覆盖：
- 自动真实交易执行
- 多币种复杂组合策略回测平台
- 面向外部客户的开放 API 商业化能力
- 高复杂度量化研究工作台

## 6. 产品范围
### 6.1 范围内
- 汇率预测机主界面
- 聊天问答区域
- 数据大盘区域
- 数据抓取与摘要 Agent
- 策略演算与预测 Agent
- 结构化预测结果展示
- 文件或中台视图中的结果沉淀

### 6.2 范围外
- 交易执行模块
- 权限体系的企业级复杂审批流
- 多租户系统
- 高阶回测引擎

## 7. 核心概念定义
### 7.1 数据抓取与摘要 Agent
负责从外部数据源或内部文件中获取信息，并输出标准化、结构化的数据摘要，供中台查看与下游预测使用。

### 7.2 策略演算 Agent
负责基于结构化数据、业务规则、指标逻辑与模型提示，生成预测判断、置信说明、关键影响因素与策略建议。

### 7.3 汇率预测结果
输出应至少包含：
- 预测对象
- 预测时间范围
- 趋势方向
- 关键依据
- 风险提示
- 数据时间戳
- 数据来源

## 8. 典型用户场景
### 场景一：研究人员查看最新市场情况
用户进入页面后，可在大盘中查看近期汇率走势、关键指标与事件摘要，再通过聊天界面提问“未来 3 天 USDCNH 偏升还是偏贬”，系统返回预测结论与主要依据。

### 场景二：中台人员查看 Agent 产出
用户在中台中查看数据抓取 Agent 的结构化输出，包括原始来源、抓取时间、摘要内容与标准字段，并可进一步查看策略演算 Agent 给出的预测结论。

### 场景三：管理者需要日报式结论
用户不关心底层计算细节，只需要看到简洁结论、核心驱动因素、风险提示和建议动作。

## 9. 功能模块设计
### 9.1 汇率预测机主页面
目标：承载聊天交互与数据大盘。

核心能力：
- 页面顶部展示产品标题、数据更新时间、当前预测币种
- 左侧或主区域展示聊天问答
- 右侧或下方展示数据大盘
- 支持查看最近一次预测摘要

验收要点：
- 页面可同时呈现问答区与数据区
- 核心信息层级清晰
- 首屏可见最新预测摘要与数据更新时间

### 9.2 聊天问答模块
目标：提供自然语言交互入口。

核心能力：
- 用户输入问题并发起预测请求
- 系统返回趋势、解释、风险与引用数据摘要
- 支持预置问题快捷入口
- 支持会话历史查看

验收要点：
- 用户能发起至少一种预测提问
- 返回结果包含结论与解释，不是只有一句话
- 会话上下文在单次页面生命周期内可保持

### 9.3 数据大盘模块
目标：让用户快速理解近期市场与预测依据。

核心能力：
- 展示汇率时间序列走势
- 展示关键指标卡片
- 展示摘要事件或驱动因子
- 展示预测结果与置信度说明

验收要点：
- 至少支持一个汇率对的趋势图
- 至少展示 3 类关键指标或摘要信息
- 指标与预测结果之间存在可理解关联

### 9.4 数据抓取与摘要 Agent
目标：完成数据采集、清洗、归档与结构化摘要。

输入来源：
- 第三方金融数据 API
- 本地导入文件
- 可选扩展资讯类数据源

核心能力：
- 拉取原始数据
- 标准化字段
- 生成摘要
- 输出结构化结果供页面与中台使用
- 记录抓取时间、来源与处理状态

验收要点：
- 数据抓取失败时有明确状态
- 结构化输出字段完整
- 同一数据源支持重复拉取和更新

### 9.5 策略演算 Agent
目标：生成可解释的预测结论。

核心能力：
- 消化结构化数据输入
- 按策略规则或模型提示生成趋势判断
- 输出方向、原因、置信说明、风险提示
- 支持生成适合中台查看的结构化卡片结果

验收要点：
- 预测结果有固定输出结构
- 至少包含方向、依据、风险三类信息
- 结果可在聊天区与中台视图复用

### 9.6 文件与中台沉淀模块
目标：将 Agent 输出沉淀为可查看、可复盘的结果。

核心能力：
- 查看历史抓取记录
- 查看历史预测记录
- 导出结构化内容或保存到文件
- 支持中台中按时间查看结果

验收要点：
- 至少能查看最近若干次输出记录
- 单次输出包含时间戳、来源与摘要

## 10. 端到端核心流程
### 10.1 数据刷新流程
1. 系统触发或用户手动触发数据抓取
2. 数据抓取 Agent 拉取并标准化数据
3. 摘要与结构化结果入库或存档
4. 页面刷新数据大盘

### 10.2 预测问答流程
1. 用户提出汇率问题
2. 系统读取最近结构化数据
3. 策略演算 Agent 生成预测结果
4. 聊天模块返回结论与解释
5. 预测记录同步沉淀到中台或文件

### 10.3 中台复盘流程
1. 用户打开历史记录
2. 查看某次抓取与预测输入输出
3. 对比数据与结论，进行复盘或校正

## 11. 信息架构建议
建议一期最小信息架构如下：
- 首页 / 汇率预测机
- 聊天会话区
- 数据大盘区
- 预测记录页
- 数据抓取记录页
- 单次记录详情页

## 12. 关键数据对象
### 12.1 汇率数据
- 币种对
- 时间
- 开盘价
- 收盘价
- 最高价
- 最低价
- 成交量或可替代指标
- 数据来源

### 12.2 摘要结果
- 摘要标题
- 摘要正文
- 标签
- 来源
- 生成时间
- 原始记录引用

### 12.3 预测结果
- 问题文本
- 预测方向
- 预测区间
- 置信说明
- 核心驱动因素
- 风险项
- 生成时间
- 关联数据版本

## 13. 权限与角色建议
一期可简化为两类角色：
- 查看角色：查看页面、图表、预测与历史记录
- 管理角色：可触发抓取、重跑预测、查看更完整的中台信息

## 14. 非功能要求
- 可追溯：每条预测需要可回溯到对应数据版本或数据时间戳
- 可解释：预测输出必须附带依据和风险提示
- 可扩展：后续可扩展多数据源与多币种
- 可观测：关键链路需要有状态和错误信息
- 可复用：Agent 输出结构应可被页面与中台共同使用

## 15. 指标与成功标准
### 15.1 一期产品验收指标
- 用户可完成至少 1 条有效预测问答
- 页面可展示至少 1 个汇率对的数据大盘
- 数据抓取 Agent 可输出结构化结果
- 策略演算 Agent 可输出结构化预测结论
- 中台或文件可查看历史记录

### 15.2 运行观察指标
- 数据抓取成功率
- 平均预测生成时延
- 页面首屏可用性
- 用户问答完成率
- 历史记录可追溯率

## 16. 版本规划建议
### V1
- 单一汇率对
- 单一外部 API 数据源
- 单页工作台
- 基础聊天问答
- 基础数据摘要与预测结论
- 历史记录查看

### V1.1
- 增加多币种支持
- 增加资讯摘要数据源
- 增强置信度说明
- 优化图表与筛选器

### V2
- 增加策略配置能力
- 增加模型版本管理
- 增加多人协作和权限细分
- 增加更完整中台工作流

## 17. 风险与依赖
### 风险
- 外部汇率数据源调用频率、质量与稳定性受限
- 预测结果容易被误解为确定性结论，需要清晰表达风险
- 初期数据维度不足时，预测解释力有限
- 若无统一结构化输出规范，两个 Agent 难以协作

### 依赖
- 至少一个可稳定访问的汇率数据源
- 可落地的结构化数据存储方案
- AI 推理或规则引擎调用能力
- 可展示图表与会话的前端容器

## 18. 需补充确认的信息
以下内容建议在进入详细设计前补齐：
- 目标币种范围是否仅限 USDCNH
- 预测时间粒度是日内、T+1、3 天还是 7 天
- 中台是否已有现成承载系统
- 文件沉淀格式要求是 Markdown、JSON、Excel 还是数据库记录
- 预测结果是否需要人工复核流程
- 是否需要接入资讯、宏观数据或事件数据
