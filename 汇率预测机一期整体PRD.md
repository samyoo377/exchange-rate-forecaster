# 汇率预测机一期整体PRD

## 1. 文档目标
本 PRD 用于定义“汇率预测机”一期产品的业务目标、用户价值、功能范围、能力边界与验收标准，指导产品、设计、研发、测试的统一协作。

### 2.1 关键信息提炼
- 已明确产品定位：预测结果用于策略辅助，不作为唯一交易依据。
- 已明确核心指标体系：RSI、Stochastic、CCI、ADX、AO、MOM。
- 已具备离线样本：Excel 内含 10min/15min 级别 OHLC 历史数据，可用于本地演示与回测验证。
- 已具备在线数据方向：Alpha Vantage 可作为主数据源。

### 2.2 当前机会点
- 机会一：先把“可解释预测”跑通，再逐步提升准确度。
- 机会二：通过“在线数据 + 离线样例”双通道提高落地稳定性。
- 机会三：通过结构化输出和历史留痕，形成可复盘能力。

## 3. 一期产品定位
构建一个面向研究和策略辅助的汇率预测工作台，支持：
- 看数据（行情与指标）
- 问趋势（聊天式问答）
- 看结论（方向、置信度、依据、风险）
- 可复盘（历史记录与数据版本追溯）
- 多图表分析（K线、指标、信号、预测结果）
- 多粒度切换（小时线、日线）

## 4. 目标用户
### 4.1 核心用户
- 外汇研究员
- 策略分析师
- 中台运营/数据分析人员

### 4.2 关键诉求
- 快速获取近期走势与技术信号
- 快速获得可解释的预测结论
- 可回看历史结论及对应依据

## 5. 一期高价值功能梳理
以下是“能尽快产生业务价值”的功能优先级排序。

### 5.1 P0（必须做）
1. **数据接入双通道**
   - 在线：Alpha Vantage 拉取 OHLC
   - 离线：Excel 导入 OHLC
2. **指标计算引擎**
   - RSI / Stochastic / CCI / ADX / AO / MOM
3. **规则融合预测引擎**
   - 生成 direction/confidence/rationale/riskNotes
4. **工作台页面**
   - 聊天问答区 + 数据大盘区 + 历史记录区
5. **多图表分析系统**
   - 价格K线图（主图）
   - 指标图（RSI、Stochastic、CCI、ADX、AO、MOM）
   - 信号事件图（买卖/中立标注）
   - 预测结果趋势图（direction 与 confidence 历史变化）
6. **小时线/日线粒度切换**
   - 支持 1H 与 1D 粒度切换
   - 切换后图表、指标、预测口径联动刷新
7. **可追溯记录**
   - 每次预测关联时间戳、数据版本、指标快照

### 5.2 P1（增强）
1. 多币对支持
2. 指标参数可配置
3. 预测结果导出（JSON/CSV）
4. 历史结果筛选与对比

### 5.3 P2（进阶）
1. 增加宏观数据与新闻事件标签
2. 规则 + 模型混合预测
3. 胜率统计与回测报告

## 6. 汇率预测机相关核心能力清单

## 6.1 数据能力
- 多源接入：API + 文件
- 数据标准化：统一字段与时间口径
- 多时间粒度管理：统一支持 1H 与 1D
- 数据质量校验：high/low/open/close 合法性检查
- 数据版本管理：每批数据可追踪

## 6.2 指标能力
- 技术指标批量计算
- 指标结果快照存储
- 指标阈值信号化（buy/sell/neutral）

## 6.3 预测能力
- 信号融合打分
- 趋势方向判定（bullish/bearish/neutral）
- 置信度生成
- 风险提示生成

## 6.4 解释与复盘能力
- 输出“结论 + 依据 + 风险”
- 历史预测回放
- 数据来源、数据版本追溯

## 6.5 产品交互能力
- 聊天式提问与响应
- 图表与指标可视化
- 刷新与重跑预测
- 历史查询和筛选

## 7. 一期范围与非目标
### 7.1 范围内
- USDCNH 单币对
- 支持小时线与日线双粒度（1H / 1D）
- 以 T+2 预测口径为主
- 基于规则的可解释预测
- 前端工作台与后端 API 打通

### 7.2 范围外
- 自动下单与交易执行
- 高频分钟级交易策略
- 复杂机器学习训练平台

## 8. 核心用户流程
### 8.1 数据刷新流程
1. 用户点击刷新或系统定时触发
2. 从 Alpha 或 Excel 拉取数据
3. 数据标准化入库
4. 自动计算指标并更新大盘

### 8.2 预测问答流程
1. 用户输入问题（例：未来 T+2 偏升还是偏贬）
2. 系统读取最新数据与指标
3. 规则引擎融合信号
4. 返回结构化预测结果
5. 结果写入历史记录

### 8.3 复盘流程
1. 用户查看历史预测
2. 展示当次数据版本、指标快照、结论与风险提示
3. 支持按时间/方向筛选

## 9. 功能需求明细

## 9.1 数据源管理
- 功能描述：管理在线与离线数据接入。
- 关键需求：
  - 在线拉取失败时，允许切换 Excel 导入。
  - 每次导入/刷新生成任务记录。

## 9.2 指标计算模块
- 功能描述：按标准公式计算指标并生成信号。
- 关键需求：
  - 指标结果可复算。
  - 对样本不足场景给出明确状态。

## 9.3 预测生成模块
- 功能描述：融合信号并生成预测结论。
- 输出结构：
  - direction
  - confidence
  - rationale[]
  - riskNotes[]
  - sourceRefs[]
  - generatedAt
  - snapshotVersion

## 9.4 工作台页面
- 功能描述：统一承载交互与展示。
- 页面要素：
  - 顶部：币对、更新时间、刷新
  - 左侧：聊天问答
  - 右侧：行情图+指标卡+预测卡
  - 底部：历史列表

## 9.5 历史记录模块
- 功能描述：沉淀预测过程，支持复盘。
- 关键需求：
  - 记录请求、结果、数据版本
  - 支持分页与筛选

## 10. 信息架构
- 首页/工作台
- 预测历史页
- 任务日志页
- 单次预测详情页

## 11. 数据与口径约束
### 11.1 标准字段
- symbol
- timestamp/tradeDate
- open/high/low/close
- source
- version

### 11.2 校验规则
- high >= low
- open/close 位于 [low, high]
- 同 symbol + 时间 + version 唯一

## 12. 非功能要求
- 可解释：每条预测必须有依据与风险。
- 可追溯：预测必须关联数据版本。
- 可观测：失败任务必须有日志。
- 可扩展：支持后续新增币对与指标。

## 13. 成功指标（一期）
### 13.1 产品验收指标
- 用户可完成至少 1 条有效预测问答。
- 页面可展示行情图、指标卡、预测卡。
- 预测结果可在历史页回查。

### 13.2 运行指标
- 数据刷新成功率
- 预测生成成功率
- 预测响应时间
- 历史追溯完整率

## 14. 版本计划
### V1（一期交付）
- 单币对 USDCNH
- Alpha + Excel 双通道
- 6 个核心指标
- 规则融合预测
- 工作台 + 历史页

### V1.1
- 多币对

### V2
- 加入事件因子
- 模型增强
- 回测与效果评估

## 15. 风险与应对
1. API 调用受限或失败
   - 应对：Excel 回退 + 缓存最近成功数据
2. 指标信号冲突
   - 应对：输出中立结论与冲突说明
3. 用户误把预测当交易指令
   - 应对：强制风险提示与免责声明

## 16. 研发对接建议
1. 先实现最小闭环，不先做复杂智能化。
2. 统一输出 schema，前后端按同一契约对接。
3. 联调优先级：健康检查 → 数据导入/刷新 → 大盘 → 预测问答 → 历史。

## 17. 待确认事项
1. 一期是否只支持 USDCNH。
2. 默认预测周期是否固定 T+2。
3. 历史记录保留策略（天数/归档）。
4. 是否需要角色权限区分（查看/管理）。

## 18. 结论
基于当前三份输入资料，一期最有价值的路线是：
- 以 Alpha Vantage 作为在线主数据，Excel 作为稳定回退
- 以规则驱动实现可解释预测
- 以工作台承载问答、大盘和历史复盘

该方案能在较低工程复杂度下快速形成可用业务价值，并为后续多因子和模型增强打下结构化基础。
